# Руководство по API для студентов

В данном документе описан контракт REST API для работы с учебными группами и студентами, учитывающий новые правила разграничения доступа для деканата.

---

## 1. Учебные Группы (Groups)
**Endpoint:** `/api/v1/protected_groups/`  
**Доступные методы:** `GET`, `POST`
**Требует авторизации:** Да

### Создание группы (POST)
**Доступ:** Только по токену Суперпользователя (`IsSuperUser`, Деканат).  
Для создания новой группы отправьте запрос:
```json
{
  "group_name": "python.02", // Обязательно (макс. 255 симв., уник.)
  "start_year": 2024,        // Обязательно
  "diploma_year": 2028       // Обязательно
}
```
*Валидации:* `start_year` <= `diploma_year`.

---

## 2. Студенты (Users)
**Endpoint:** `/api/v1/protected_users/`  

### Регистрация Сотрудника Деканата (Djoser)
При создании пользователя через базовый Djoser endpoint `POST /api/v1/auth/users/`, аккаунт автоматически получает права Суперпользователя (`is_superuser=True`). Такие пользователи играют роль сотрудников деканата.

### Создание Студента Деканатом (POST `/api/v1/protected_users/`)
**Доступ:** Только по токену Суперпользователя (`IsSuperUser`).  
При создании все реквизиты доступа указываются вручную:
```json
{
  "username": "student_login", // Строго обязательно!
  "password": "secure_password", // Строго обязательно!
  "email": "student@example.com", // Строго обязательно!
  "first_name": "John",           // Опционально
  "last_name": "Doe",             // Опционально
  "user_group": 1,                // Опционально, ID группы
  "date_of_birth": "2000-01-01",
  "phone": "89991234567",         // Опционально (Regex: ^8\d{10}$)
  "tg_name": "@john_doe",         // Опционально
  "is_student": true
}
```

### Редактирование Данных Студентом (PUT / PATCH)
Студент не может менять свои системные доступы. Поля `username`, `password`, `email` полностью **ИСКЛЮЧЕНЫ** из контракта обновления.
*Принимаются только поля:* `first_name`, `last_name`, `user_group`, `date_of_birth`, `phone`, `tg_name`.

### Профили (GET)
Ответ листинга пагинирован (DRF Standard). `email` перемещен в блок контактов, а также добавлены прикрепленные фотографии (`photos`):
```json
{
  "id": 1,
  "username": "student_login",
  "first_name": "John",
  "last_name": "Doe",
  // ...
  "contacts": {
    "email": "student@example.com",
    "phone": "89991234567",
    "tg_name": "@john_doe"
  },
  "photos": [
    {
      "id": 5,
      "url": "media/pictures/1.jpg",
      "mime": "jpg",
      "size": "2M"
    }
  ],
  "avatars": [
    {
      "id": 3,
      "picture_string": "data:image/png;base64,...",
      "is_active": true
    }
  ]
}
```

### Массовый импорт студентов (CSV)
**Endpoint:** `/api/v1/protected_users/import_students/`  
**Метод:** `POST`  
**Доступ:** Только Суперпользователь (Деканат)  
**Формат запроса:** `multipart/form-data`. Ожидается файл в поле `file` или `csv`.

**Требования к CSV-файлу:**
- Разделитель: точка с запятой `;`
- Кодировка: UTF-8
- Первая строка: заголовки (названия колонок не важны, важен их строгий порядок).

**Строгий порядок колонок:**
1. ID Группы (Опционально. Если пусто — студент не попадает в группу)
2. Логин (`username`) (Строго обязательно, уник.)
3. Пароль (Строго обязательно)
4. Email (Строго обязательно, уник. формат)
5. Имя (`first_name`) (Опционально)
6. Фамилия (`last_name`) (Опционально)
7. Дата рождения (`YYYY-MM-DD`, Опционально)
8. Телефон (Regex `^8\d{10}$`, Опционально)
9. Telegram (До 24 симв., Опционально)
10. Статус студента (`True`/`False`, Опционально, по дефолту `True`)

> [!NOTE]
> **Автоматический Rollback:** Импорт полностью атомарный. Это значит, что если хотя бы 1 строка содержит ошибку валидации (неверный телефон, дубль логина), весь процесс завершится отказом. База останется невредимой, а в теле возвращенного JSON (HTTP 400) будет находиться подробный массив `details` с расшифровкой, на каких именно студентах (по индексу массива) произошел сбой.

## 3. Выгрузка отчетов (Без пагинации)
Применяют те же фильтры, что и GET-листинг.
- **CSV**: GET `/api/v1/protected_users/csv_report/`
- **XLS**: GET `/api/v1/protected_users/xls_report/`
