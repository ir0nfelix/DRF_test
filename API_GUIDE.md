# Руководство по API для студентов

В данном документе описан контракт REST API для работы с учебными группами и студентами, включая получение списков (листинги) и операции управления (создание, редактирование, удаление).

---

## 1. Учебные Группы (Groups)
**Endpoint:** `/api/v1/protected_groups/`  
**Доступные методы:** `GET`, `POST`
**Требует авторизации:** Да

### Листинг и Детали (GET)
При запросе списка или детальной информации о группе, возвращаются следующие поля:
- **`id`** *(integer)*: Уникальный идентификатор группы в базе данных.
- **`group_name`** *(string)*: Название группы по учебному плану (макс. 255 символов, уникальное).
- **`count`** *(integer)*: Текущее количество привязанных к группе студентов.
- **`start_year`** *(integer)*: Год начала обучения группы.
- **`diploma_year`** *(integer)*: Год запланированного окончания обучения (выпуска).
- **`is_archive`** *(boolean)*: Возвращает `true`, если обучение уже закончено.
- **`students`** *(array)*: (Доступно только при детальном GET запросе группы по ID) Список студентов.

**Фильтрация и Сортировка листинга:**
- Фильтры: `group_name`, `search`, `is_archive`, `start_year`, `stop_year`, `count` (поддерживают суффиксы `__gt`, `__gte`, `__lt`, `__lte`).
- Сортировка (`?ordering=`): `group_name`, `students_count`, `start_year`, `diploma_year`.

### Создание группы (POST)
Для создания новой группы отправьте `POST` запрос с телом:
```json
{
  "group_name": "python.02", // Обязательно (макс. 255 симв., уник.)
  "start_year": 2024,        // Обязательно
  "diploma_year": 2028       // Обязательно
}
```
*Валидации:* `start_year` не может быть больше `diploma_year`.

## 2. Студенты (Users)
**Endpoint:** `/api/v1/protected_users/`  
**Доступные методы:** `GET`, `POST`, `PUT`, `PATCH`, `DELETE`
**Требует авторизации:** Да

### Листинг и Детали (GET)
Ответ листинга пагинирован (DRF Standard). Результаты(`results`) содержат:
- **`id`** *(integer)*: Уникальный идентификатор базы.
- **`username`, `email`, `first_name`, `last_name`** *(string)*: Базовые профильные данные.
- **`user_group`** *(integer/null)*: ID привязанной учебной группы.
- **`group_name`** *(string/null)*: Имя привязанной академической группы.
- **`date_of_birth`** *(string)*: Дата рождения (формат `YYYY-MM-DD`).
- **`is_currently_study`** *(boolean)*: Студент считается активным, если он не отчислен и группа не в архиве.
- **`contacts`** *(object)*: Вложенный объект с контактами: `{"phone": "...", "tg_name": "..."}`.

**Фильтрация и Сортировка листинга:**
- Фильтры: `user_group`, `is_student`, `date_of_birth` (`__lt` и т.д.), `first_name`, `last_name`, `group_name`, `search`, `is_currently_study`, `has_contacts`.
- Сортировка (`?ordering=`): `first_name`, `last_name`, `date_of_birth`, `user_group__group_name`.

### Управление студентом (POST / PUT / PATCH)
Запись контактов происходит в корне JSON в плоском формате. Пароль генерируется системой автоматически и не требуется в запросе.
```json
{
  "username": "student_login", // Обязательно, уникально
  "email": "student@example.com", // Обязательно (корректный email формат)
  "first_name": "John",           // Опционально
  "last_name": "Doe",             // Опционально
  "user_group": 1,                // Опционально, ID группы
  "date_of_birth": "2000-01-01",  // Строго YYYY-MM-DD
  "phone": "89991234567",         // Опционально (Regex: ^8\d{10}$)
  "tg_name": "@john_doe",         // Опционально (макс. 24 символа)
  "is_student": true              // Опционально
}
```

### Удаление студента (DELETE)
Физическое удаление пользователя по ID (например `/api/v1/protected_users/3/`).

## 3. Выгрузка отчетов (Без пагинации)
Применяют те же фильтры, что и GET-листинг.
- **CSV**: GET `/api/v1/protected_users/csv_report/`
- **XLS**: GET `/api/v1/protected_users/xls_report/`
