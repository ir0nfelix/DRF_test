# Руководство по API для студентов

В данном документе описан обновленный контракт REST API для получения списков (листингов) учебных групп и студентов.

---

## 1. Листинг Учебных Групп
**Endpoint:** `/api/v1/protected_groups/`  
**Метод:** `GET`  
**Требует авторизации:** Да

### Структура ответа (Поля контракта)
При запросе списка или детальной информации о группе, возвращаются следующие поля:
- **`id`** *(integer)*: Уникальный идентификатор группы в базе данных.
- **`group_name`** *(string)*: Название группы по учебному плану (например, `программирование.01`).
- **`count`** *(integer)*: Вычисляемое поле, отображающее текущее количество привязанных к группе студентов.
- **`start_year`** *(integer)*: Год начала обучения группы.
- **`diploma_year`** *(integer)*: Год запланированного окончания обучения (выпуска).
- **`is_archive`** *(boolean)*: Вычисляемое проперти. Возвращает `true`, если обучение уже закончено (то есть показатель `diploma_year` строго меньше текущего календарного года).

### Фильтрация (GET-параметры)
Листинг групп поддерживает мощную систему фильтрации. Можно передавать несколько параметров одновременно (через символ `&`):

1. **Строковый поиск по названию**
   - `group_name=значение` — ищет по частичному или полному совпадению (без учета регистра). *Пример: `?group_name=дизайн`*
   - `search=значение` — работает аналогично, под капотом использует DRF `SearchFilter`.
2. **Фильтр по статусу архива**
   - `is_archive=true / false` — автоматическое маппирование на сверку года окончания с текущим.
3. **Фильтрация по датам (Год начала и Выпуск)**
   Доступно точное вхождение и использование суффиксов операторов сравнения (`__gt` — больше, `__gte` — больше или равно, `__lt` — меньше, `__lte` — меньше или равно).
   - `start_year` (например, `?start_year__gte=2022&start_year__lte=2024`)
   - `stop_year` (маппируется на базу данных как `diploma_year`, пример: `?stop_year=2028`)
4. **Фильтрация по количеству студентов**
   - `count` — поддержка точного значения и суффиксов `__gt, __gte, __lt, __lte`. *Пример: `?count__gt=10`*

### Сортировка (GET-параметры)
Управляется через параметр `?ordering=`. Для сортировки по убыванию необходимо добавить знак минуса `-` в начало.
Доступные поля сортировки:
- `?ordering=group_name` (По алфавиту имени)
- `?ordering=students_count` (По количеству студентов. Обратите внимание: параметр фильтра называется `count`, а сортировки — `students_count`!)
- `?ordering=start_year` (По дате поступления)
- `?ordering=diploma_year` (По дате выпуска)

## 2. Листинг Студентов
**Endpoint:** `/api/v1/protected_users/`  
**Метод:** `GET`  
**Требует авторизации:** Да

### Пагинация (Standard Django Paginator)
Ответ листинга студентов разделен на страницы. В корне JSON-ответа находятся поля:
- **`count`** *(integer)*: Общее количество студентов, удовлетворяющих запросу.
- **`next`** *(string/null)*: URL на следующую страницу (если есть).
- **`previous`** *(string/null)*: URL на предыдущую страницу (если есть).
- **`results`** *(array)*: Массив объектов студентов для текущей страницы (структура описана ниже).

Параметры пагинации (GET-параметры):
- `?page=N` — номер запрашиваемой страницы
- `?page_size=N` — желаемое количество результатов на странице (до 100, по умолчанию 10)

### Структура ответа (Поля контракта в блоке `results`)
- **`id`** *(integer)*: Уникальный идентификатор базы.
- **`username`, `email`, `first_name`, `last_name`** *(string)*: Базовые профильные данные.
- **`user_group`** *(integer/null)*: ID привязанной учебной группы.
- **`group_name`** *(string/null)*: Имя привязанной академической группы (маршрутизируется автоматически для удобства чтения).
- **`date_of_birth`** *(string, формат `YYYY-MM-DD`)*: Обязательная дата рождения.
- **`is_currently_study`** *(boolean)*: Вычисляемое поле-сводка. Студент считается активным (`true`), только если он не отчислен (внутреннее поле `is_student=True`) И его группа продолжает учебу (`is_archive=False`).
- **`contacts`** *(object)*: Вложенный JSON-блок с контактными данными студента.
  - Возвращаемый формат: `{"phone": "89991234567", "tg_name": "durov"}`
  - Если какие-то контакты отсутствуют, возвращается пустая строка. Одно из полей (или оба) всегда присутствует в словаре.

### Фильтрация (GET-параметры)
По студентам доступна мощная система фильтрации и поиска:

1. **Строгое соответствие и диапазоны**
   - `user_group=ID` — отфильтровать студентов по ID академической группы
   - `is_student=true / false` — статус отчисления по документам
   - `date_of_birth` — поиск по датеописания рождения (например, `?date_of_birth=2000-01-01`). Доступны суффиксы для диапазонов: `__gt`, `__gte`, `__lt`, `__lte` (например, `?date_of_birth__gte=2003-01-01`).

2. **Поиск по тексту (Частичное совпадение)**
   - `first_name=значение` — поиск по имени (без учета регистра)
   - `last_name=значение` — поиск по фамилии (без учета регистра)
   - `group_name=значение` — поиск по названию группы (например `?group_name=программирование`)
   - `search=значение` — глобальный поиск DRF, ищет совпадения в `first_name`, `last_name` и `group_name`.

3. **Булевые смарт-фильтры (Вычисляемые)**
   - `is_currently_study=true / false` — фильтрует студентов, которые не отчислены (`is_student=True`) и чья группа всё ещё учится.
   - `has_contacts=true / false` — фильтрует студентов, у которых заполнен хотя бы один контакт (телефон или Telegram).

### Сортировка (GET-параметры)
Управляется через параметр `?ordering=`. Для обратного порядка используйте минус `-` (например `?ordering=-date_of_birth`). Доступные поля:
- `first_name`
- `last_name`
- `date_of_birth`
- `user_group__group_name`

### Выгрузка отчетов (Экспорт в CSV и XLS)
Данные листинга можно скачать в виде файлов, используя специальные маршруты (actions). Выгрузка автоматически применяет все переданные вами параметры фильтрации и поиска с основного листинга (например `?last_name=Ivanov&ordering=first_name`). При передаче таких GET-параметров в выгрузку попадет именно отфильтрованная выборка. Обе выгрузки не подвержены пагинации и выгружают всех найденных студентов.

- **Выгрузка в CSV (.csv)**
  **Endpoint:** `/api/v1/protected_users/csv_report/`
  **Метод:** `GET`
  Движок возвращает файл (MIME `text/csv`) с BOM-меткой для корректного отображения кириллицы в MS Excel без дополнительных конвертаций, разделитель — точка с запятой `;`.

- **Выгрузка в XLS (.xls)**
  **Endpoint:** `/api/v1/protected_users/xls_report/`
  **Метод:** `GET`
  Движок возвращает реальный бинарный файл `MS-Excel` (MIME `application/ms-excel`).
