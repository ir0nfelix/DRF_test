# Практическая работа: Ролевая работа с порталом студентов (DRF)

В этом задании вы продемонстрируете полный цикл работы с порталом: от создания учетных записей сотрудников и массовой загрузки данных, до обновления профиля со стороны обычного студента. 

## Часть 1: Подготовка окружения (Деканат)

### Шаг 1: Создание сотрудника деканата
Сотрудники деканата обладают правами суперпользователя. Такие права выдаются автоматически при регистрации через базовый user-эндпоинт.
- **Действие:** Выполните запрос на создание сотрудника деканата. Схема регистрации:
  - **Метод:** `POST /api/v1/auth/users/`
  - **Тело (JSON):** `{"username": "hexlet_officer", "password": "secure123", "email": "officer@hexlet.ru"}`
- Запишите свои логин и пароль в блокнот.

> **Теория:** АПИ, доступные **только** для сотрудников деканата:
> - `POST /api/v1/protected_groups/` (ручное создание учебных групп)
> - `POST /api/v1/protected_users/` (ручное создание студента)
> - `POST /api/v1/protected_users/import_students/` (массовый импорт студентов из CSV-файла)

### Шаг 2: Авторизация как сотрудником деканата
- **Действие:** Залогиньтесь под созданным сотрудником.
  - **Метод:** `POST /api/v1/auth/login/`
  - **Тело (JSON):** `{"username": "hexlet_officer", "password": "secure123"}`
- Полученный Access (JWT) токен сохраните в блокнот и вставьте в авторизацию среды (Postman/ThunderClient).

### Шаг 3: Создание учебной группы
- **Метод API:** `POST /api/v1/protected_groups/` (требует токен из Шага 2).
- **Название группы:** Формат — `ФамилияСтудента.06-23ИСИП`
- Выполните запрос и запишите `id` созданной группы.


## Часть 2: Массовый импорт данных (Batch CSV)

### Шаг 4: Загрузка списка студентов из файла
Нам нужно быстро загрузить целую группу. Гораздо удобнее сделать это через файл конфигурации CSV.
> **Немного теории об CSV:**
> CSV (Comma-Separated Values) — популярный текстовый формат для представления табличных данных. Каждая строка файла — это одна строка таблицы. Значения колонок разделяются специальным символом, в нашем случае это точка с запятой (`;`). Десятки тысяч строк в CSV весят копейки, файл легко открывается в Excel, Google Sheets или в простом блокноте, его легко сгенерировать любым скриптом.
> 
> **Почему валидация БД отклоняет ВЕСЬ файл при малейшей ошибке?** В нашем API реализована строгая «базовая атомарность» (`transaction.atomic()`). Если хотя бы 1 строчка из 100 содержит дублирующийся `email` (уже есть в базе) или неправильно написанный телефон, транзакция разрушится, и база откатит добавления назад. Это защищает систему от «грязных» вставок и дублей при повторных отправках формы.

- **Действие:**
  1. Откройте файл-образец группы по ссылке (*преподаватель устно выдаст ссылку на Google SpreadSheets или файл в чате*).
  2. Скопируйте из  файла 5 строк со студентами в  блокнот, а в общем файле залейте использованные строки красным цветом (чтобы одногруппники не использовали ваших студентов и не было ошибки уникальности логинов по вашей вине!).
  3. В скачанном вами блокноте-файле (`my_import.csv`) в первом столбце (до первой `;`) пропишите **ID вашей созданной группы** из Шага 3. (Было `;user1;...`, стало `7;user1;...`).
  4. Загрузите файл.
- **Спецификация API загрузки (Массовый импорт):**
  - **URL:** `POST /api/v1/protected_users/import_students/`
  - **Auth:** Токен сотрудника деканата
  - **Body Format:** `multipart/form-data`
  - **В Postman:** Вкладка Body -> Переключить на form-data -> Ключ (Key) назвать `file` -> Тип ключа выбрать File (справа в поле Key появится стрелочка) -> Прикрепить ваш созданный `my_import.csv`.
- Убедитесь, что сервер вернул код `201 Created` и импортировал объекты.

### Шаг 9: Ручное добавление опоздавшего (Деканат)
